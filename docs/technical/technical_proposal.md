# LoongClaw-Railway-Forecast 技术方案

## 一、项目背景与目标

### 1.1 研究背景

在全球气候变暖背景下，极端降水事件频发，对铁路沿线小流域的防灾安全构成严峻挑战。传统短临降水预报方法存在以下问题：

- 数值模式分辨率不足，难以捕捉小尺度对流系统
- 统计外推方法对强对流发展演变预测能力有限
- 缺乏与水文模型的有效耦合，难以实现致灾风险预警

### 1.2 研究目标

构建"降水预报-径流模拟-风险推演"一体化智能预警系统：

1. **短临降水预报**：实现 0-3 小时高分辨率（1km）降水预报
2. **致灾风险推演**：耦合 SCS-CN 水文模型，评估铁路沿线灾害风险
3. **系统集成**：开发预警平台原型，支撑业务化应用

### 1.3 预期成果

- 核心期刊论文 1-2 篇
- 开源代码库
- 预警平台原型
- 技术验证报告

### 2.2 关键技术

#### 2.2.1 MambaSwin-UNet-STA 模型

**设计思路：**

融合三种先进架构的优势：
- **Mamba**：线性复杂度的序列建模，适合长时序依赖
- **Swin Transformer**：层次化空间特征提取，计算高效
- **时空注意力（STA）**：显式建模时空交互

**网络结构：**

```python
class MambaSwinUNetSTA(nn.Module):
    """
    编码器:
    - MambaBlock: 时序特征提取 (B, T, C, H, W) → (B, T, C', H, W)
    - SwinBlock: 空间特征提取 (B, T, C', H, W) → (B, T, C'', H/2, W/2)
    
    瓶颈层:
    - SpatioTemporalAttention: 时空注意力融合
    
    解码器:
    - 上采样 + 跳跃连接
    - 多尺度特征聚合
    
    输出:
    - (B, forecast_steps, H, W) 降水预报
    """
```

**参数配置：**

| 参数 | 值 | 说明 |
|------|-----|------|
| base_channels | 64 | 基础通道数 |
| time_steps | 6 | 输入时间步 |
| forecast_steps | 6 | 预报时间步 |
| mamba_d_state | 16 | Mamba 状态维度 |
| swin_window_size | 8 | Swin 窗口大小 |
| sta_num_heads | 8 | 注意力头数 |

#### 2.2.2 LoongClaw 动态路由框架

**设计思路：**

单一模型难以兼顾所有场景，采用混合专家（MoE）思想，动态选择最优预测策略。

**四大专家：**

| 专家 | 能力 | 适用场景 |
|------|------|----------|
| AI专家 | 深度学习预测 | 数据充足、模式稳定 |
| 物理专家 | PDE约束、质量守恒 | 物理一致性要求高 |
| 统计专家 | ARIMA、光流外推 | 短时平稳、数据稀疏 |
| LLM专家 | 知识推理、异常解释 | 极端事件、不确定性高 |

**路由机制：**

```python
class DynamicRouter(nn.Module):
    """
    输入: 当前气象状态特征
    输出: 专家权重分布
    
    路由策略:
    - 学习型路由: 神经网络预测权重
    - 规则型路由: 基于阈值规则
    - 混合路由: 结合学习与规则
    """
```

#### 2.2.3 物理约束模块

**质量守恒约束：**

$$\frac{\partial P}{\partial t} + \nabla \cdot (P \vec{v}) = 0$$

**地形强迫：**

$$w = u \frac{\partial h}{\partial x} + v \frac{\partial h}{\partial y}$$

其中 $h$ 为地形高度，$w$ 为地形诱导的垂直速度。

**COTREC 光流外推：**

基于 Lucas-Kanade 方法计算光流场，采用半拉格朗日方案进行外推。

#### 2.2.4 SCS-CN 水文模型

**径流计算：**

$$Q = \frac{(P - I_a)^2}{P - I_a + S}$$

其中：
- $Q$: 地表径流 (mm)
- $P$: 降水量 (mm)
- $I_a = 0.2S$: 初始抽损
- $S = 25400/CN - 254$: 潜在最大滞留量

**CN 值确定：**

根据土地利用类型、土壤类型、前期土壤湿润条件查表确定。

**风险等级划分：**

| 等级 | 径流量阈值 (mm) | 风险描述 |
|------|-----------------|----------|
| 0 | < 10 | 无风险 |
| 1 | 10-25 | 低风险 |
| 2 | 25-50 | 中风险 |
| 3 | 50-100 | 高风险 |
| 4 | > 100 | 极高风险 |

## 三、数据方案

### 3.1 数据来源

| 数据类型 | 来源 | 时间范围 | 空间分辨率 |
|----------|------|----------|------------|
| ERA5再分析 | ECMWF | 2004-2024 | 0.25° |

| 雨量计 | 兰州铁路局 | 2004-2024 | 站点 |
| DEM | SRTM | - | 30m |

### 3.2 数据处理流程

```
原始数据
    ↓
质量控制 (缺失值检测、异常值剔除)
    ↓
时空匹配 (插值到统一网格)
    ↓
特征工程 (地形特征、雷达特征、时序特征)
    ↓
标准化 (Min-Max / Z-Score)
    ↓
训练/验证/测试集划分
```

### 3.3 特征工程

**地形特征：**
- 坡度 (Slope)
- 坡向 (Aspect)
- 汇流面积 (Flow Accumulation)
- 地形湿度指数 (TWI)

**雷达特征：**
- 组合反射率 (Composite Reflectivity)
- 回波顶高 (Echo Top)
- 垂直积分液态水 (VIL)

**时序特征：**
- 滑动统计 (Rolling Mean/Max/Std)
- 变化趋势 (Trend)
- 周期特征 (Hourly/Seasonal)

## 四、训练方案

### 4.1 训练配置

| 参数 | 值 |
|------|-----|
| 框架 | PyTorch Lightning |
| 优化器 | AdamW |
| 学习率 | 1e-4 |
| 调度器 | OneCycleLR |
| Batch Size | 16 |
| Epochs | 200 |
| 混合精度 | FP16 |
| 梯度裁剪 | 1.0 |

### 4.2 损失函数

**组合损失：**

$$L = L_{MSE} + 0.5 L_{weighted} + 0.3 L_{focal} + 0.2 L_{SSIM} + 0.1 L_{smooth}$$

- $L_{MSE}$: 基础均方误差
- $L_{weighted}$: 加权MSE（高降水权重更高）
- $L_{focal}$: Focal Loss（解决类别不平衡）
- $L_{SSIM}$: 结构相似性损失（保持空间结构）
- $L_{smooth}$: 平滑损失（时空连续性）

### 4.3 评估指标

**分类指标（按阈值）：**
- TS (Threat Score)
- POD (Probability of Detection)
- FAR (False Alarm Rate)
- ETS (Equitable Threat Score)

**连续指标：**
- MAE (Mean Absolute Error)
- RMSE (Root Mean Square Error)
- 空间相关系数

### 4.4 数据增强

- 水平/垂直翻转
- 90°/180°/270° 旋转
- 高斯噪声注入
- 随机裁剪
